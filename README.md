# Claude Code Dojo 🥋

Claude Code を**実務で使いこなせる**ようになるための、自学型ハンズオン教材です。
ミッション形式で、基礎対話からカスタマイズ（CLAUDE.md / skills / hooks）、MCP連携、並列開発、CI連携までを一気通貫で体験します。

## この教材の方針

機能の知識より「**頼み方**」と「**検証の習慣**」を重視します。

- 各ミッションに「良い頼み方 / 悪い頼み方」の比較があります
- Claude の変更は **diff とテストで自分が確認してから受け入れる**——これを全ミッションで徹底します
- ミッション03では**わざと失敗**してもらいます。指示の質が結果を決めることを体で覚えるためです

## 構成

```
docs/        ← ミッション00〜11（この順に進める）
app/         ← 題材アプリ（勤怠集計CLI + API。バグが仕込んであります）
solutions/   ← 模範解答（詰まってから見る）
```

## 進め方

> **前提環境**: WSL2（Ubuntu）/ macOS / Linux ＋ Node.js v20 以上。詳細は [docs/00-setup.md](docs/00-setup.md) を参照。Windows で WSL 未導入の方は [docs/appendix-wsl.md](docs/appendix-wsl.md) から始めてください

1. [docs/00-setup.md](docs/00-setup.md) から順に進める
2. 各ミッションの**チェックポイント**を満たしたら次のミッションへ
3. 詰まったら15分は自力（Claudeに聞くのは自力に含む！）、それでもダメなら solutions を見る

## レベル別ロードマップ

| レベル | ミッション | 到達目標 |
|---|---|---|
| 🥉 初心者 | 00 → 01 → 02 → 03 | 頼み方の3原則と検証の習慣が身につく。実務でコード読解・バグ修正に使える |
| 🥈 中級者 | 04 → 05 → 06 → 07 → 08 | CLAUDE.md・セッション運用・skills・hooks・permissions で自分の環境を育て、守れる |
| 🥇 上級者 | 09 → 10 → 11 | MCP・並列開発・CI連携でチームの生産性に貢献できる |

- 経験者はミッション01〜02を流し読みで進めてOK。ただし**ミッション03は全員必修**です
- [docs/cheatsheet.md](docs/cheatsheet.md) は修了後も手元に置いて使ってください

## 勉強会で使う場合

章単位で抽出できます。

- **入門会（90分）**: 01 + 02 + 03（頼み方と検証の習慣）
- **コンテキスト管理会（45分）**: 04 + 05（コンテキストの原則と、それを守るセッション操作）
- **カスタマイズ会（90分）**: 04 + 06 + 07（CLAUDE.md / skills / hooks）
- **セキュリティ会（60分）**: 08（permissions と秘密情報の保護）
- **応用会（90分）**: 09 + 10（MCP / 並列開発）

## 注意事項

- 会社のコード・データを題材に試す前に、チームのセキュリティポリシーを確認してください
- `app/.env` は教材用のダミー値です。実案件の .env は絶対にコミットしないでください
- ミッション11（CI連携）は個人の GitHub リポジトリで試すこと
- 題材アプリのバグは**わざと**です。教材を進める前に直さないでください（直すのがミッションです）
