# ミッション00: 環境構築

## ゴール

- Claude Code をインストールし、ログインして対話できる状態にする
- 題材アプリのテストが（1件失敗した状態で）動くことを確認する
- 権限モードの意味を理解する

## 背景解説

Claude Code はターミナルで動く AI コーディングエージェントです。ChatGPT のようなチャット AI との最大の違いは、**あなたのマシン上でファイルを読み書きし、コマンドを実行できる**ことです。「このバグ直して」と頼むと、コードを読み、修正し、テストを実行して確認するところまで自分でやります。

だからこそ「何をどこまで許可するか」（権限モード）が重要になります。ここは後述します。

## 前提の確認

### 必要なもの

- WSL2（Ubuntu）または macOS / Linux
- Node.js v20 以上（`node -v` で確認。入っていなければ nvm でインストール）
- git の初歩（`clone` / `status` / `diff` が何をするか分かればOK）

> **gitが不安な人へ**: このDojoではgit操作を自分で打つ場面はほぼありません。困ったら **Claude Code に「git操作もお願い」と頼んでOK** です（例:「今の変更を確認したいから diff を見せて」）。それも立派な使い方です。

## ミッション手順

### 1. Claude Code のインストール

```bash
npm install -g @anthropic-ai/claude-code
claude --version
```

### 2. ログイン

```bash
claude
```

初回起動時にブラウザ認証が求められます。会社アカウントの指定がある場合はチームのガイドに従ってください。

### 3. 題材アプリのセットアップ

このリポジトリの `app/` に移動して依存をインストールします。

```bash
cd app
npm install
npm run seed     # サンプルデータ投入
npm test         # ← 1件失敗するのが正常です（ミッション2で直します）
npm run lint     # ← こちらは通ります
```

### 4. 権限モードを知る

Claude Code はツール（ファイル編集・コマンド実行）を使う前に確認を求めてきます。起動後に `Shift+Tab` でモードを切り替えられます。

| モード | 挙動 | 使いどころ |
|---|---|---|
| 通常 | 編集・コマンドのたびに確認 | 最初はこれ。何をしようとするか観察できる |
| auto-accept | 編集は自動承認 | 信頼できる作業を任せるとき |
| plan mode | 読み取り専用。計画だけ立てる | 大きな変更の前の相談 |

**このDojoでは、まず通常モードで「Claudeが何をしようとしているか」を毎回読むこと**から始めてください。承認の前に内容を読む癖が、後の「検証の習慣」につながります。

## 発展課題

- `claude --help` を眺めて、気になるオプションを1つ試す
- `/help` をClaude Code内で実行し、スラッシュコマンドの一覧を見る

## ハマりどころ

| 症状 | 対処 |
|---|---|
| `npm install -g` で権限エラー | nvm 経由の Node.js を使う（システムの Node に sudo で入れない） |
| better-sqlite3 のビルドエラー | Node v20 以上か確認。`node -v` が古い場合は `nvm use 20` |
| 会社プロキシで認証が通らない | チームの環境構築ガイドを参照（プロキシ設定が必要な場合あり） |
| テストが「全部」通ってしまう | それはこの教材のバグです。作成者に報告してください |
