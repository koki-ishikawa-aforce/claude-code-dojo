# ミッション08: サブエージェントと並列作業

## ゴール

- サブエージェント（Task/Agent）に調査を委譲し、メイン会話のコンテキストを守る使い方を覚える
- git worktree で複数の Claude Code を並列に走らせる体験をする

## 背景解説

### サブエージェントとは

Claude Code は自分の「分身」を起動して作業を委譲できます。分身は**独立したコンテキスト**を持ち、結果の要約だけを親に返します。

これが効くのはこんな場面です:

- **大規模な調査**: 「このリポジトリ全体から○○の使用箇所を洗い出して」→ 分身が大量のファイルを読み、親の会話には結論だけが返る（親のコンテキストが汚れない）
- **独立した複数タスク**: それぞれ別の分身に並列で任せる

ミッション04で学んだ「コンテキストは有限」の応用編です。**大量に読む作業は分身にやらせ、親は判断に集中する**。

### git worktree による並列作業

サブエージェントとは別の並列化として、**Claude Code を複数枚同時に走らせる**方法があります。同じブランチで2つの Claude が同じファイルを触ると壊れるので、git worktree で作業ディレクトリごと分離します。

```bash
git worktree add ../dojo-feature-a feature-a   # 別ディレクトリに別ブランチを展開
```

ターミナルを2枚開き、それぞれのディレクトリで `claude` を起動すれば、機能Aと機能Bを完全並列で進められます。

## ミッション手順

### 1. 調査をサブエージェントに委譲する

`app/` で:

```
サブエージェントを使って、このリポジトリ全体で toMinutes 関数が
使われている箇所をすべて調査して。呼び出し元と用途を一覧にまとめて。
```

観察ポイント: エージェントの起動表示が出て、しばらく後に**要約された結果だけ**が返ってきます。あなたの会話にはファイルの中身の垂れ流しが残っていません。

### 2. 委譲すべきかを判断する練習

次の2つを Claude に頼む前に、「サブエージェント向きか？」を自分で予想してから頼んでください:

```
A. src/cli.ts の switch 文を読みやすくして
B. このプロジェクトの全ファイルを調べて、エラーハンドリングの方針が
   統一されていない箇所を洗い出して
```

（答え: A は直接編集で十分。B は読む量が多いので委譲向き）

### 3. worktree で並列開発を体験する

このリポジトリで、2つの独立した小タスクを並列でやってみます。

```bash
cd ~/claude-code-dojo
git worktree add ../dojo-csv -b feature/csv-export
git worktree add ../dojo-holiday -b feature/holiday
```

- ターミナル1: `cd ../dojo-csv/app && claude` → 「summary コマンドに CSV 出力オプションを追加して」
- ターミナル2: `cd ../dojo-holiday/app && claude` → 「土日の日付を登録しようとしたら警告する機能を追加して」

両方が完了したら、それぞれで `npm test` を確認し、worktree を片付けます。**未コミットの変更が残っていると `git worktree remove` はエラーになる**ので、変更を残すなら先にコミット、実験を捨てるなら `--force` を付けます:

```bash
git worktree remove --force ../dojo-csv       # 実験を捨てる場合（変更を残すなら先にコミットして --force なしで）
git worktree remove --force ../dojo-holiday
git branch -D feature/csv-export feature/holiday   # 実験を捨てる場合
```

### 良い頼み方 / 悪い頼み方

> ❌ 並列させたい2タスクを同じセッションで交互に指示する（コンテキストが混ざり、両方の質が下がる）
>
> ✅ 独立したタスクは worktree + 別セッションに分離する。1セッション1タスクの徹底

## ハマりどころ

| 症状 | 対処 |
|---|---|
| サブエージェントを使ってくれない | 「サブエージェントを使って」と明示するか、タスクを「全体を調査して要約して」の形にする |
| worktree で npm エラー | worktree ごとに `npm install` が必要（node_modules はコピーされない） |
| `worktree remove` が modified or untracked files エラー | 未コミットの変更が残っている。コミットするか、捨てるなら `--force` を付ける |
| 並列した変更が衝突した | タスクの切り方の問題。触るファイルが重ならない単位で分けるのがコツ |
| 分身の結果が信用できない | 親と同じ。要約の根拠（ファイル名:行）を出させて自分で1箇所は確認する |
