git diff（未コミットの変更）を対象にコードレビューを行う。

## 手順

1. `git diff` で変更内容を取得する（ステージ済みも見る場合は `git diff HEAD`）
2. 以下の観点でレビューする
   - バグ・ロジックの誤り（最優先）
   - SPEC.md との整合性
   - テストの有無・妥当性
   - 命名・可読性
3. 指摘は重要度順に、日本語で報告する
4. 問題がなければ「指摘なし」と明言する

## 出力形式

- 各指摘: `[高/中/低] ファイル名:行数 — 指摘内容と修正案`
- 最後に一言サマリー（マージしてよいか/修正が必要か）
